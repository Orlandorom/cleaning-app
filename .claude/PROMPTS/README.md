# Prompts Directory

This directory contains prompts for AI-Driven Development (AIDD).

## Naming Convention

- `phase-{N}-{name}.md` — For new development phases
- `feature-{name}.md` — For specific feature implementations
- `fix-{issue-description}.md` — For bug fixes
- `refactor-{description}.md` — For refactoring tasks
- `task-{description}.md` — For one-off tasks

## Prompt Structure

Each prompt should include:

1. **Objective** — What needs to be built/fixed/changed
2. **Context** — Relevant background from PROJECT_CONTEXT.md
3. **Requirements** — Specific acceptance criteria
4. **Files to modify** — List of files with brief description
5. **Validation** — How to verify the work is correct
